AI-DLC

AI-Driven Development Life Cycle

An adaptive, decision-driven workflow that scales with project complexity — keeping AI focused and humans in control.

🔄 Adaptive Workflow

Automatically scales from simple straight-line paths to multi-unit incremental delivery based on complexity.

🎯 Decision Gates

Every phase asks the right questions before generating artifacts. You decide, AI executes.

📋 Manifest Tracking

Pause, resume, and roll back across sessions. Never lose progress.

🚀 Full Lifecycle

From context assessment through deployment — one workflow covers inception, construction, and operation.

⚡ Parallel Execution

Tasks dispatched in dependency waves via sub-agents with isolated file ownership for safe concurrency.

🌐 Multi-Platform

Works on Kiro (IDE and CLI) and Claude Code. Same skills, same workflow, any environment.

The Problem with AI Coding Assistants

Powerful but undirected. Without structure, the results are unpredictable.

Without AI-DLC

  • Inconsistent architecture across sessions
  • Technology choices that don't align with your project
  • Skipped edge cases and missing validation
  • No way to pause, resume, or roll back
  • Lost context after long conversations
  • No audit trail — can't trace why decisions were made

With AI-DLC

  • Structured decisions validated for conflicts
  • You choose the stack — AI executes your choices
  • EARS acceptance criteria catch edge cases early
  • Manifest tracks state — resume anytime
  • Context recovery from steering files
  • Full audit trail — every decision logged

Three Phases

Every project flows through Inception, Construction, and Operation.

Inception

Understand the Problem

Context assessment, requirements gathering, decomposition into deliverable units.

Construction

Solve the Problem

Technology decisions, detailed design, task planning, code implementation.

Operation

Ship and Run

Final build verification, CI/CD pipeline generation, deployment configuration.

Workflow Paths

The workflow adapts based on project complexity.

Simple Project

≤ 5 stories, 1 domain, 1 user type

  • Context
  • Requirements D1
  • Design D3
  • Tasks D4
  • Implement
  • Build & Test
  • Deploy D5

Complex Project

5+ stories, 2+ domains, multiple user types

  • Context
  • Requirements D1
  • Decomposition D2
  • ↳ For each unit:
  • Design D3
  • Tasks D4
  • Implement ⟳
  • Build & Test
  • Deploy D5
D1–D5 Decision gate — you answer questions before AI generates

Decision Gates

At each phase, the AI generates questions with multiple options. You choose. It validates for conflicts.

GatePhaseWhat You Decide
D1RequirementsScope, user types, entities, integrations, business rules
D2DecompositionArchitecture pattern, unit boundaries, dependencies, sequence
D3DesignTech stack, frameworks, database, testing, code organization
D4TasksBreakdown strategy, TDD vs test-after, granularity
D5DeployCI/CD platform, environments, deployment strategy, rollback

Conflict Detection

🔴 Critical: "Microservices with solo developer"
🟡 Major: "Prisma + MongoDB limited support"
🟢 Minor: "Enterprise observability for MVP"

Skip Option

Say "use recommendations" to auto-fill, or "skip validation" to proceed without conflict checks.

Skills Structure

Core skills drive the workflow. Support skills are invoked on demand.

Core Skills (workflow phases)

  • aidlc — orchestrator (routes, status, rollback)
  • aidlc-context — scans workspace, produces context
  • aidlc-requirements — user stories with EARS criteria
  • aidlc-decomposition — breaks into units (DDD)
  • aidlc-design — architecture, components, API spec
  • aidlc-tasks — sequenced tasks with execution waves
  • aidlc-implement — writes code from task specs
  • aidlc-build — runs build, tests, quality gates
  • aidlc-deploy — generates CI/CD pipeline configs

Support Skills (on demand)

  • aidlc-prototype — throwaway spike to validate ideas
  • aidlc-reverse-engineer — deep codebase analysis (13 reports)
  • aidlc-solutions-review — cross-unit design review
  • aidlc-code-review — code quality review

Each skill contains

  • SKILL.md — compact index loaded on dispatch
  • actions/ — step-by-step instructions
  • assets/ — output templates
  • references/ — catalogs & guides
